# Hilaal Documentation

## Purpose

Hilaal is a crescent-visibility explorer. It compares how the first lunar crescent changes
by location, Gregorian date, local time, map projection, and published visibility criterion.

It is not an official moon-sighting authority. Weather, haze, extinction, local horizon profile,
observer acuity, optical aid, and jurisdictional rules are outside the static public model.

## Public Site Modes

The repository has two surfaces:

- `streamlit run app.py`: local live-calculation mode. This can calculate arbitrary coordinates
  and times with Skyfield.
- `pages/`: static public site mode. This is what is embedded at
  `abdullahhussain.com.au/hilaal/`. It is generated by
  `scripts/build_pages_site.py` because GitHub Pages cannot run Python/Skyfield at request time.

The public site precomputes a full Gregorian month, every hour across each selected
calendar date, selected city/location rows, global latitude/longitude grid points in 15 degree
steps, country representative points, state/province sample points, capital city points, and major
city points.

## Active Visibility Criteria

Only real formula-based criteria are shown in the public dropdown for now:

- Yallop q criterion.
- Odeh V criterion.

The earlier Composite, Ilyas, Shaukat, and SAAO entries have been removed from the public app
until they can be represented as real sourced implementations rather than legacy heuristics.

## Astronomy Engine

Astronomy values are calculated with Skyfield and the DE421 ephemeris. The engine uses
topocentric geometry: the Sun and Moon are calculated as seen from the observer's latitude,
longitude, and elevation, not just from Earth's centre.

For each observer and UTC instant, the app computes:

- Sun altitude.
- Sun azimuth.
- Moon altitude.
- Moon azimuth.
- Moon-Sun elongation.
- ARCV.
- DAZ.
- Moon phase angle.
- Moon illumination fraction.
- Topocentric Moon semidiameter.
- Crescent width W.
- Moon age since the latest astronomical new moon/conjunction.
- Moon distance.
- Sun distance.
- Sunrise/sunset.
- Moonrise/moonset.
- Sunset-to-moonset lag.
- Previous astronomical new moon/conjunction, labelled as Moon birth.
- Next astronomical new moon/conjunction.

## Definitions Of Astronomy Values

### Moon Altitude

Moon altitude is the apparent height of the Moon above or below the local horizon. Positive values
mean the Moon is above the geometric horizon; negative values mean it is below it.

### Sun Altitude

Sun altitude is the apparent height of the Sun above or below the local horizon. Crescent
visibility normally improves after sunset as the sky darkens, but the crescent must still remain
above the horizon.

### Elongation

Elongation is the angular separation between the apparent Moon and Sun positions. A very small
elongation means the crescent is too close to the Sun's glare.

### ARCV

ARCV means arc of vision. It is the Moon altitude minus the Sun altitude, in degrees:

```text
ARCV = moon_altitude - sun_altitude
```

It measures the vertical separation between Moon and Sun. Yallop and Odeh both use ARCV.

### DAZ

DAZ means difference in azimuth. It is the absolute horizontal angular difference between Moon and
Sun, in degrees. DAZ is useful for understanding whether the Moon is horizontally separated from
the Sun even when vertical separation is small.

### Crescent Width W

The app estimates topocentric crescent width in arcminutes as:

```text
W = SD * (1 - cos(phase_angle))
```

Where:

- `SD` is the topocentric apparent semidiameter of the Moon in arcminutes.
- `phase_angle` is the lunar phase angle from Skyfield.

Yallop and Odeh use W alongside ARCV.

### Moon Illumination

Moon illumination is the fraction of the lunar disc lit by the Sun. For very young crescents this
value is small, but illumination alone is not enough; altitude, sky brightness, ARCV, and W also
matter.

### Moon Age

Moon age is the time since the latest astronomical new moon/conjunction. The app labels that
conjunction as Moon birth. Moon age is useful context, but the real visibility criteria shown here
are based on ARCV and W rather than age alone.

### Moon Lag

Moon lag is:

```text
moon_lag_minutes = moonset - sunset
```

Positive lag means the Moon sets after the Sun. Negative lag means the Moon sets before sunset.

## Islamic Sighting Geometry Gates

Many Islamic crescent-sighting workflows require at least these local-date geometry facts before a
new-crescent sighting claim is meaningful:

1. The Moon must have been born before local sunset.
2. The Moon must set after local sunset, so it is geometrically above the horizon for some interval
   after sunset.

The app displays:

```text
Moon birth UTC
Next moon birth UTC
Sunset
Moonset
Moon lag minutes
Moon age at sunset
Born before sunset
Moonset after sunset
Geometry gate
```

`Geometry gate` is true only when conjunction happened before sunset and moonset happens after
sunset. This is not a fatwa, calendar decision, or official declaration.

## Yallop Criterion

The Yallop implementation uses:

```text
q = (ARCV - (11.8371 - 6.3226W + 0.7319W^2 - 0.1018W^3)) / 10.3741
```

Where:

- `ARCV` is the Moon-Sun altitude difference in degrees.
- `W` is topocentric crescent width in arcminutes.

Yallop bands:

| Score | Category | q range |
| --- | --- | --- |
| 6 | A: Easily visible | q > 0.216 |
| 5 | B: Visible under perfect conditions | -0.014 < q <= 0.216 |
| 4 | C: May need optical aid | -0.160 < q <= -0.014 |
| 3 | D: Will need optical aid | -0.232 < q <= -0.160 |
| 2 | E: Not visible with telescope | -0.293 < q <= -0.232 |
| 1 | F: Only photographic | -0.490 < q <= -0.293 |
| 0 | G: Not visible | q <= -0.490 |

## Odeh Criterion

The Odeh implementation uses:

```text
V = ARCV - (-0.1018W^3 + 0.7319W^2 - 6.3226W + 7.1651)
```

Where:

- `ARCV` is the Moon-Sun altitude difference in degrees.
- `W` is topocentric crescent width in arcminutes.

Odeh bands:

| Score | Category | V range |
| --- | --- | --- |
| 3 | A: Easily visible by naked eye | V >= 5.65 |
| 2 | B: Visible by optical aid, may be naked-eye | 2.00 <= V < 5.65 |
| 1 | C: Visible only with optical aid | -0.96 <= V < 2.00 |
| 0 | D: Not visible | V < -0.96 |

## Maps And Point Types

The public site renders three map modes:

- Best On Selected Date: each point is shaded by its best visibility band over the selected UTC
  date from 00:00 through the precomputed hourly samples for that date. Tooltips show the UTC hour
  that produced the best band.
- Selected Date + Time: visibility at the selected date/time instant. This is the restrictive
  "where would the crescent be visible at this exact moment" mode.
- Date + Time Across Lat/Lon: grid points near the selected latitude and longitude are emphasized,
  with latitude and longitude sweep charts shown below the map.

Map calculation points include:

- latitude/longitude grid points at 15 degree intervals,
- representative points for countries from Natural Earth,
- state/province sample points,
- capital city points,
- and major city points.

Country points are representative interior points, not official observation locations.

## Calendar

The visibility-across-date view is a Gregorian month calendar. Each day is shaded by the best
visibility band found across the precomputed 24-hour day for the selected location and selected
criterion.

## Time Zones

The public site separates calculation time from display time:

- The selected date and time are interpreted as the selected location's local civil time.
- Map calculations are evaluated at the corresponding UTC instant.
- Event display can be shown in selected-location time, browser-local time, or UTC.

The site uses the browser's date, time, and time zone for initial controls where possible.
Browser geolocation is used only when the user presses `Use current` and grants permission.

## Search And Address Matching

The public static site has an offline search list for bundled cities, countries, states, capitals,
and aliases. It also asks the Photon OpenStreetMap geocoder for live address suggestions when a
typed query is not already an offline match. Selecting a live suggestion uses that latitude and
longitude as the map marker and uses the nearest precomputed map sample for static chart values.

Because the public site is static, it does not run a full timezone lookup service for arbitrary
addresses. For live address suggestions, the calculation-time offset defaults to the browser's
current UTC offset. Preset locations use their bundled UTC offset.

For precise current position, use the `Use current` button and grant browser location permission.

## Projections

The map includes:

- Plate Carree: simple rectangular longitude/latitude.
- Robinson: compromise world projection.
- Mollweide: equal-area world projection.
- Equal Earth: modern equal-area world projection.
- Orthographic: globe-like view centered on the selected location.

Every projection distorts something: shape, area, distance, or direction.

## Limitations

- Public data is precomputed for the generated Gregorian month.
- Public time sampling is hourly.
- Weather, cloud, haze, extinction, local terrain, and actual horizon obstructions are not modelled.
- Live address suggestions depend on the public Photon/OpenStreetMap geocoder being reachable.
- Arbitrary address time zones currently use the browser's UTC offset.
- Country and state points are representative map points, not official national or regional
  sighting locations.
